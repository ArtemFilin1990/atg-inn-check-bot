# Error recovery checklists

## ENV_FAIL
- Проверить обязательные ENV в README и коде.
- Убедиться, что пустые/невалидные значения обрабатываются до старта.
- Для секрета указывать `[[TBD]]`, не подставлять фиктивные значения в код.

## DEP_FAIL
- Сверить `requirements.txt`/lock-файл с импортами.
- Переустановить зависимости и повторить команду.
- Зафиксировать версию, если поломка от несовместимого обновления.

## NETWORK_FAIL
- Проверить DNS/доступность хоста/timeout.
- Повторить запрос с минимальным воспроизведением.
- Если внешний сервис недоступен, пометить как ограничение среды.

## TEST_FAIL
- Запустить исходную команду, затем более подробный прогон (`pytest -vv`).
- Локализовать один падающий тест/файл.
- Применить минимальный фикс и повторно прогнать исходную команду.

## RUNTIME_FAIL
- Собрать минимальный сценарий старта + healthcheck.
- Сохранить traceback/логи и точку падения.
- Исправить root cause, а не симптомы.
