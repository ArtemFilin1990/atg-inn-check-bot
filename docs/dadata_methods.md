# Методы DaData, используемые в проекте

Документ фиксирует только **подтверждённые в коде репозитория** методы и параметры API DaData.

## Что в проекте соответствует странице `https://dadata.ru/api/find-party/`

В клиенте проекта логика «поиска компании» реализована двумя REST-методами DaData:

1. `suggest/party` — первичный поиск по строке.
2. `findById/party` — уточнение/получение детальной карточки по идентификатору.

Именно эта последовательность используется в `find_party_universal()` (сначала `suggest`, затем `findById`, при необходимости fallback).

## Проверка относительно `hflabs/dadata-py`

- В текущем проекте **не используется SDK `dadata-py`**: в зависимостях отсутствует пакет `dadata`, интеграция выполнена напрямую через `httpx`.
- Поэтому Source of Truth для интеграции — `app/dadata_client.py` и тесты клиента.

## 1) `suggest/party`

- URL: `https://suggestions.dadata.ru/suggestions/api/4_1/rs/suggest/party`
- Назначение: поиск компании по текстовому запросу (название, ИНН, ОГРН).
- Используется в функции `suggest_party()` и в первом шаге универсального поиска `find_party_universal()`.

### Параметры (подтверждённые)

- `query: str` — строка поиска (обязательный).
- `count: int` — количество подсказок (обязательный, `> 0`).

### Валидация/ограничения в проекте

- Пустой `query` запрещён.
- `count <= 0` запрещён.

## 2) `findById/party`

- URL: `https://suggestions.dadata.ru/suggestions/api/4_1/rs/findById/party`
- Назначение: получение карточки юрлица по идентификатору (ИНН/ОГРН), а также уточнение результатов после `suggest/party`.
- Используется в `find_by_id_party()` и как второй шаг в `find_party_universal()`.

### Параметры (подтверждённые)

- `query: str` — идентификатор компании (обязательный).
- `count: int` — число результатов (обязательный, `> 0`).
- `branch_type: str | None` — опционально.
- `kpp: str | None` — опционально.
- `type` (в коде аргумент `entity_type`) — опционально.

### Валидация/ограничения в проекте

- Пустой `query` запрещён.
- `count <= 0` запрещён.

## 3) Общие правила вызова DaData в проекте

- Авторизация: заголовок `Authorization: Token <DADATA_API_KEY>`.
- Таймаут HTTP-запроса: `10s`.
- Кеширование: TTL cache, до `512` записей, TTL `900s` (15 минут).
- Формат ответа: ожидается JSON-объект (`dict`).

## 4) Алгоритм универсального поиска

Функция `find_party_universal()` реализует следующую последовательность:

1. Нормализует ввод пользователя и определяет тип (`inn` / `ogrn` / `name`).
2. Всегда сначала вызывает `suggest/party`.
3. Если есть подсказки — пытается взять лучший `inn/ogrn` и уточняет через `findById/party`.
4. Если подсказок нет и исходный тип был числовым (`inn`/`ogrn`) — делает fallback в `findById/party`.
5. Если уточнение не дало данных — возвращает результат `suggest/party`.

## 5) Ограничения верификации в текущем окружении

Прямой доступ к `dadata.ru` и `github.com` из контейнера ограничен proxy (HTTP 403), поэтому документ основан на исходниках и тестах этого репозитория, без добавления неподтверждённых полей/параметров.
